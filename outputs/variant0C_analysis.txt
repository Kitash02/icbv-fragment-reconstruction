OBJECTIVE
---------
Prevent false positive upgrades by adding appearance gating AFTER ensemble voting.
If ensemble suggests MATCH but appearance signals are marginal, downgrade to WEAK_MATCH.

IMPLEMENTATION
--------------
File: src/ensemble_postprocess_variant0C.py

Gating Rule (added after line 113 in original ensemble_postprocess.py):
```python
if ensemble_verdict == 'MATCH':
    # Require strong appearance match to confirm upgrade
    if bc_color < 0.75 or bc_texture < 0.70:
        # Appearance is marginal → downgrade back to WEAK_MATCH
        ensemble_verdict = 'WEAK_MATCH'
```

This prevents the ensemble from upgrading WEAK_MATCH → MATCH when:
- Color similarity (Bhattacharyya) < 0.75, OR
- Texture similarity (LBP Bhattacharyya) < 0.70

The thresholds are set to catch cross-source pairs that might have good geometry
but weak appearance (different pottery sources, different pigments, etc.)

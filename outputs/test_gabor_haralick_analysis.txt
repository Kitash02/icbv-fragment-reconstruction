================================================================================
ROOT CAUSE ANALYSIS
================================================================================

WHY GABOR + HARALICK FAILED:

1. TEXTURE OVERGENERALIZATION:
   - Pottery surfaces share common micro-texture patterns:
     * Clay grain structure (particle size ~0.1-1mm)
     * Firing effects (thermal gradients, oxidation)
     * Surface weathering (erosion, patina)
   - Gabor filters capture these periodic patterns
   - Haralick captures the spatial statistics
   - BUT: These patterns are GENERIC to all pottery, not artifact-specific

2. HIGH CROSS-SIMILARITY:
   - Different pottery pieces have similar:
     * Grain orientation (aligned by manufacturing)
     * Frequency content (determined by clay composition)
     * Spatial statistics (GLCM values cluster for ceramics)
   - Result: Bhattacharyya coefficient BC > 0.85 for most pairs
   - Geometric mean penalty: (0.85)^0.25 = 0.96
   - Exponential penalty: (0.96)^4.0 = 0.85
   - Net effect: Only 15% score reduction (insufficient)

3. FEATURE WEIGHT IMBALANCE:
   - Current formula: geometric_mean(color, LBP, Gabor, Haralick)^4
   - When all 4 features score high, penalty becomes weak
   - Color alone was more discriminative (different pigments)
   - Adding Gabor+Haralick DILUTED the color signal

================================================================================

IMMEDIATE ACTIONS:

1. REMOVE Gabor and Haralick features from pottery matching
   - They provide negative value for this specific domain
   - Generic texture features cannot discriminate pottery sources

2. INCREASE color feature weight
   - Color (pigment chemistry) is the most discriminative signal
   - Current weight: 0.25 (in geometric mean with 3 other features)
   - Recommended: 0.6-0.8 (or use color as primary + others as secondary)

3. ADJUST penalty formula
   - Current: geometric_mean^4 (dilutes strong signals)
   - Recommended: weighted_sum or min(signals) for veto logic
   - Allow any single low-similarity feature to reject a match

ALTERNATIVE APPROACHES:

1. Use Gabor/Haralick ONLY for same-artifact matching
   - They excel at positive detection (100% accuracy)

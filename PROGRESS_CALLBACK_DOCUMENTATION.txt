PROGRESS CALLBACK IMPLEMENTATION - SUMMARY
==========================================

This document describes the optional progress_callback functionality added to src/main.py.

## Overview

Optional progress reporting has been added to the pipeline WITHOUT changing core logic.
The callback system is completely optional - when not provided, the code behaves identically
to the original implementation.

## Modified Functions

### 1. extract_contours(fragment_paths, run_logger, progress_callback=None)

**Purpose**: Extract contours from fragment images with progress reporting

**Progress Reports**:
- After each fragment: "Loading fragments... (N/TOTAL)" with percentage
- Example: "Loading fragments... (3/8)" at 37%

**Code Sample**:
```python
if progress_callback:
    progress_callback(f"Loading fragments... ({idx + 1}/{total})",
                    percent=int((idx / total) * 100))
```

### 2. compute_compatibility_matrix(all_segments, all_pixel_segs, images, names, run_logger, progress_callback=None)

**Purpose**: Compute pairwise compatibility scores with progress reporting

**Progress Reports**:
- Start: "Computing compatibility scores..." at 0%
- End: "Computing compatibility scores... (100%)" at 100%

**Code Sample**:
```python
if progress_callback:
    progress_callback("Computing compatibility scores...", percent=0)

# ... actual computation ...

if progress_callback:
    progress_callback("Computing compatibility scores... (100%)", percent=100)
```

### 3. run_relaxation_labeling(compat_matrix, run_logger, progress_callback=None)

**Purpose**: Run relaxation labeling with progress reporting

**Progress Reports**:
- Start: "Relaxation iteration 0/50..." at 0%
- End: "Relaxation iteration N/50..." at 100% (N = actual iterations)

**Code Sample**:
```python
if progress_callback:
    progress_callback("Relaxation iteration 0/50...", percent=0)

# ... actual relaxation ...

n_iters = len(trace)
if progress_callback:
    progress_callback(f"Relaxation iteration {n_iters}/{MAX_ITERATIONS}...",
                    percent=100)
```

### 4. run_pipeline(args, progress_callback=None)

**Purpose**: Main pipeline orchestrator, now accepts optional progress callback

**Additional Progress Reports**:
- "Extracting contours..." (no percentage)
- "Rendering results..." (no percentage)

**Code Sample**:
```python
def run_pipeline(args: argparse.Namespace, progress_callback=None) -> None:
    # ... existing code ...

    images, contours, all_segments, all_pixel_segs, names = extract_contours(
        fragment_paths, run_logger, progress_callback
    )

    if progress_callback:
        progress_callback("Extracting contours...")

    # ... more processing ...
```

## Callback Format

All callbacks follow this signature:

```python
def progress_callback(message: str, percent: int = None) -> None:
    """
    Parameters
    ----------
    message : str
        Human-readable progress message
    percent : int, optional
        Progress percentage (0-100), or None if not applicable
    """
    pass
```

## Example Usage

### CLI Usage (Unchanged)
```bash
python src/main.py --input data/sample --output outputs/results --log outputs/logs
```

No progress callbacks are invoked when using CLI (backward compatible).

### Programmatic Usage with Progress Reporting
```python
import argparse
from main import run_pipeline

def my_progress_handler(message, percent=None):
    if percent is not None:
        print(f"[{percent}%] {message}")
    else:
        print(f"[INFO] {message}")

args = argparse.Namespace(
    input='data/sample',
    output='outputs/results',
    log='outputs/logs'
)

run_pipeline(args, progress_callback=my_progress_handler)
```

Output would be:
```
[12%] Loading fragments... (1/8)
[25%] Loading fragments... (2/8)
[37%] Loading fragments... (3/8)
...
[INFO] Extracting contours...
[0%] Computing compatibility scores...
[100%] Computing compatibility scores... (100%)
[0%] Relaxation iteration 0/50...
[100%] Relaxation iteration 23/50...
[INFO] Rendering results...
```

## Key Design Principles

1. **Optional by Default**: All progress_callback parameters default to None
2. **No Logic Changes**: Core algorithm logic is completely unchanged
3. **Backward Compatible**: CLI usage works exactly as before
4. **Consistent Format**: All callbacks use (message, percent=None) signature
5. **Strategic Placement**: Callbacks only at meaningful progress points
